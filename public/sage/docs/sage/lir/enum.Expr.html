Variants§

§

Annotated(Box<Self>, Annotation)

An expression along with data about its source code location. This is used for error reporting.

§

ConstExpr(ConstExpr)

A constant expression.

§

Many(Vec<Self>)

A block of expressions. The last expression in the block is the value of the block.

§

Declare(Box<Declaration>, Box<Self>)

Declare any number of variables, procedures, types, or constants. Evaluate a sub-expression in that scope.

§

While(Box<Self>, Box<Self>)

Create a while loop: while the first expression evaluates to true, evaluate the second expression.

§

If(Box<Self>, Box<Self>, Box<Self>)

An if-then-else expression.

Evaluate a condition. If the condition is true, evaluate the first expression. Otherwise, evaluate the second expression.

§

When(ConstExpr, Box<Self>, Box<Self>)

A constant, compile time if-then-else expression.

Evaluate the condition as a constant expression. If the condition is true, then this when expression is replaced with the first expression. Otherwise, this when expression is replaced with the second expression.

§

Match(Box<Self>, Vec<(Pattern, Self)>)

A match expression.

§

IfLet(Pattern, Box<Self>, Box<Self>, Box<Self>)

An if-let expression.

This attempts to match an expression against a pattern, and if it succeeds, evaluates the body expression with the matched variables in scope. If the match fails, the else expression is evaluated.

§

UnaryOp(String, Box<Self>)

Perform a unary operation on two expressions.

§

BinaryOp(String, Box<Self>, Box<Self>)

Perform a binary operation on two expressions.

§

TernaryOp(String, Box<Self>, Box<Self>, Box<Self>)

Perform a ternary operation on three expressions.

§

AssignOp(String, Box<Self>, Box<Self>)

Perform an assignment operation on two expressions.

§

Refer(Mutability, Box<Self>)

Reference this expression (i.e. get a pointer to it).

§

Deref(Box<Self>)

Dereference this expression (i.e. get the value it points to).

§

DerefMut(Box<Self>, Box<Self>)

Store an expression to an address (a pointer).

§

Apply(Box<Self>, Vec<Self>)

Apply a function with some arguments.

§

Return(Box<Self>)

Return a value from a function.

§

Array(Vec<Self>)

An array of expressions.

§

Tuple(Vec<Self>)

A tuple of expressions.

§

Union(Type, String, Box<Self>)

A union: a collection of named fields. The Type value is the type of the union. The String field is the field the union is being initialized with. The Box<Self> field is the value of the field we want to initialize.

§

EnumUnion(Type, String, Box<Self>)

A tagged union: a typechecked union of different variants. The Type value is the type of the tagged union. The String field is the variant the tagged union is being initialized with. The Box<Self> field is the value of the union’s data we want to initialize.

§

Struct(BTreeMap<String, Self>)

A structure of fields to expressions.

§

As(Box<Self>, Type)

Cast an expression to another type.

§

Member(Box<Self>, ConstExpr)

Get a field or member from a structure, union, or tuple. For tuples, use an Int constant expression to access the nth field (zero indexed). For unions or structures, use a Symbol constant expression to access the field.

Do NOT instantiate this directly: use the .field method on the container expression.

§

Index(Box<Self>, Box<Self>)

Index an array or pointer with an expression that evaluates to an Int at runtime.

pub fn let_var( var: impl ToString, mutability: impl Into<Mutability>, t: Option<Type>, expr: impl Into<Self>, ret: impl Into<Self> ) -> Self

Create a let binding for an expression.

This will create a new scope with the variable var defined. var will be declared with the type t, and the expression e will be assigned to it.

The result of this expression is the ret value, which is evaluated under this new scope.

When this expression is finished evaluating, var will be removed from the scope.
